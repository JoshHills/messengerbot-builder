Prerequisites

Installation

This generator produces a Javascript file as a NodeJS webhook that must be securely hosted and accessed by the Messenger API. Follow the set-up guide, and we'll do the bulk of the work.

Here are some libraries I depend on:

  • Express - npm install express
  • Messengerbot - npm install messenger-bot

If you're a bit lost, here are some useful resources:

  • Node Package Manager
  • Let's Encrypt
  • NGINX

Using the generator

Building upon the messenger API, this generator provides an easy interface to build message elements in response to user triggers. Different kinds of triggers have been provided, along with multiple different templates for response - I support generic and button bubble templates; each may be duplicitous, and all fields abide by the standards set forth in the Messenger documentation.

To provide dynamic functionality without having to delve into the generated code, I provided pre-code boxes in case you need to access external APIs. Code written in these boxes will be added as-is. Furthermore, if you want to reference any variables or dynamically set the textual content of other elements, anything wrapped in { } will be parsed as functional code. There are alrady some variables provided, and more may be accessed if you understand the messenger-bot library...

  • {text} - gets the sender's message contents
  • {first_name} - gets the sender's first name
  • {first_name} - gets the sender's last name
  • {postback} - gets the postback id (if within postback trigger) New

Precedence is determined by the order in which you add triggers!

Required fields are denoted with asterisks, but no checking is provided - you may want to be able to generate the file without them; be warned however that this may break things!

Feel free to contact me me if you'd like to leave some feedback or suggest some changes.

What does this mean?

Adding a welcome message displays something to the user from your chatbot's page before they've sent their first message. It's a good opportunity to introduce somebody to the functionality your chatbot provides, and the set the tone of discussion.

What does this mean?

You can provide your bot with a fall-back response for when the user does not trigger a command. It'll help make your bot easier to use, and prevent it from seeming unresponsive or broken.
